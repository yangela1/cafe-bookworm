# Agent Instructions: Cafe Bookworm

Welcome! You are assisting in the development of **Cafe Bookworm**, a cafe review platform for Metro Vancouver. This document provides the guidelines and standards you must follow.

## 🎨 Design Principles
- Please use a clean and minimal design style, only add shadows if prompted, no glassmorphism effect.
- **Micro-animations**: Implement hover effects, scale transitions, and smooth loading states.
- **Modern Typography**: Use Inter font.
- **DaisyUI Integration**: Leverage DaisyUI components but customize them with custom CSS/Tailwind classes
- **Dynamic & Alive**: The interface should feel responsive and interactive.

## 🛠 Tech Stack
- **Framework**: Next.js (App Router)
- **Logic**: TypeScript
- **Styling**: Tailwind CSS + DaisyUI
- **Data**: Static mock data (for now) in `data/`

## 🏗 Project Structure
- `app/`: Next.js pages and layouts.
- `app/_components/`: Reusable UI components.
- `data/`: Mock data files and TypeScript types.
- `public/`: Static assets (images, icons).

## 📋 Interaction Guidelines
1. **Plan Before You Act**: For significant changes, create/update `implementation_plan.md` and wait for approval.
2. **Use Artifacts**: Track progress in `task.md` and summarize changes in `walkthrough.md`.
3. **SEO First**: Implement descriptive title tags, meta descriptions, and semantic HTML (H1-H6) on every page.
4. **Clean Code**: Maintain documentation integrity, preserve existing comments, and follow TypeScript best practices.

## 🚀 Quality Checklist
- [ ] Responsive on all screen sizes?
- [ ] Accessibility (ARIA labels, semantic tags) followed?
- [ ] Fast performance and optimized images?
